name: review
description: Review content or changes with structured feedback
---

Review what is specified in $ARGUMENTS:
- A file path: review that file
- "changes" or "diff": review uncommitted changes
- "branch": review current branch vs main

## Review Checklist

### Content (text, proposals, reports)
- [ ] Voice -- does it match `admin/VOICE-PROFILE.md`?
- [ ] Source requirements -- does every ESG/CSRD claim have a traceable reference?
- [ ] No greenwashing -- are claims supported by actual data?
- [ ] Core message -- does the reader get the point in the first two sentences?
- [ ] No forbidden phrases (see writing-standards.md)

### Technical Accuracy (CSRD/ESG content)
- [ ] Correct directive references (article numbers, delegated acts)
- [ ] Thresholds and timelines are current
- [ ] Sector-specific requirements accurately stated
- [ ] No conflation of CSRD with other frameworks (GRI, TCFD) without clarification

### Structure
- [ ] Files in the right place in project structure
- [ ] No duplicated information
- [ ] Agent files reference source material instead of copying it

## Output

Organize findings by severity:

### Must Fix
Issues that block delivery or contain errors.

### Should Fix
Improvements that raise quality.

### Suggestions
Optional enhancements.

### Well Done
Things that work well.

End with summary and recommendation.
